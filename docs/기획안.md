# Receipt Reporter — 출장 영수증 자동 보고서 생성기

## 1. 프로젝트 개요

출장 후 책상 위에 쌓이는 영수증 사진/PDF를 한꺼번에 업로드하면, OCR로 금액·통화·일자·가맹점을 인식하고 카테고리를 자동 분류해 **부가세 매입세액 공제 가능 여부까지 판정한 정산 보고서**(Excel/Word/PDF)를 만들어주는 로컬 FastAPI 도구다. 12개 통화·17개 비용 카테고리·국내/해외 출장 모두 대응한다. 클라우드 OCR 비용을 피하기 위해 EasyOCR을 로컬에서 돌린다.

## 2. 만든 이유 (Problem)

- 출장 후 영수증 정리·환산·카테고리 분류·세무 정합성 점검을 사람이 수동으로 하면 1회 출장당 1~3시간이 소요된다.
- **부가세 매입세액 공제** 판단(접대비/해외/면세/적격증빙 여부) 로직을 매번 수기로 적용하면 누락·오공제가 발생해 세무 리스크가 생긴다.
- 다국가 출장은 통화별 환율 적용·소수점 처리·국가별 영수증 양식 차이 때문에 더 까다롭다.
- 클라우드 OCR(Google Vision·AWS Textract 등)은 1,000장에 5~15달러로 누적되면 부담이고, 영수증에 적힌 거래처 정보가 외부로 나가는 점도 부담스럽다.

## 3. 솔루션

| 영역 | 결정 사항 |
|---|---|
| OCR 엔진 | EasyOCR (로컬, GPU/CPU 모두 가능) — 외부 전송 0건 |
| 다국어 | 통화 선택에 따라 OCR 언어팩 자동 전환 (KRW→ko, JPY→ja, CNY→ch_sim 등 12개 매핑) |
| 입력 | 이미지(jpg/png/bmp) + PDF, 한 번에 최대 50건/10MB |
| 분류 | 17개 비용 카테고리(교통·숙박·식비·접대·통신·등록·업무용품·기타) 자동 분류 |
| 세무 검증 | 적격증빙(세금계산서/현금영수증/카드매출전표) 판별 + 부가세 공제 가능 여부 룰 엔진 |
| 출력 | Excel(시트별 분류) / Word(문서형) / PDF(인쇄용) 3종 동시 생성 |
| 세션 | `data/sessions.json`으로 세션 영속화 — 작업 중단 후 이어서 작업 가능 |
| UI | FastAPI + Jinja2 HTML — 브라우저로 로컬 접속 (`http://127.0.0.1:8500`) |

## 4. 핵심 기능

### 4.1 영수증 업로드 → OCR → 파싱
- 이미지/PDF 다중 업로드 (드래그 앤 드롭)
- 통화 선택 → 그에 맞는 OCR 언어팩 자동 로드
- `parser.py`가 OCR 결과에서 가맹점·일자·총액·부가세·결제수단을 추출

### 4.2 자동 분류 (17개 카테고리)

| 코드 | 카테고리 | 코드 | 카테고리 |
|---|---|---|---|
| T01 | 항공료 | M01 | 식비 |
| T02 | 철도/버스 | M02 | 음료/간식 |
| T03 | 택시 | E01 | 접대비 |
| T04 | 렌터카 | E02 | 선물/기념품 |
| T05 | 자가용/주유 | C01 | 통신비 |
| T06 | 대중교통 | R01 | 등록/참가비 |
| A01 | 호텔 | S01 | 업무용품 |
| A02 | 기타 숙박 | O01 | 비자/여권/보험 |
| | | O99 | 기타 |

### 4.3 세무 검증 (부가세 공제 자동 판정)

```
해외 출장 → 전액 불공제
적격증빙 없음(SIMPLIFIED/NONE) → 불공제
접대비 카테고리(E*) → 불공제
부가세 0원 → 불공제 (면세 거래)
그 외 → 공제 가능
```

적격증빙 임계금액: 일반 30,000원 / 접대비 10,000원

### 4.4 보고서 출력 (3종 동시)
- **Excel**: 카테고리별 시트 분리 + 합계 시트 + 공제 가능/불가 분리
- **Word**: 결재 양식 출력용 문서
- **PDF**: 첨부 영수증 썸네일 포함 인쇄본

## 5. 데이터 흐름

```
[브라우저 업로드]
    ↓
FastAPI /upload → 파일 저장 (data/uploads)
    ↓
OCREngine.recognize() (EasyOCR, 통화별 언어 자동 전환)
    ↓
ReceiptParser.parse() (가맹점/일자/금액/부가세/결제수단 추출)
    ↓
categorizer.calculate_summary() (카테고리 분류 + 적격증빙 + 공제 판정)
    ↓
ExcelReportGenerator / WordReportGenerator / PDFReportGenerator
    ↓
data/output/{session_id}.{xlsx,docx,pdf}
```

## 6. 기술 스택

- 백엔드: Python 3.10+, FastAPI, Uvicorn
- OCR: EasyOCR (PyTorch 기반, CPU/GPU 옵션)
- 파싱: 정규식 + 룰 기반 휴리스틱
- 보고서: openpyxl(Excel), python-docx(Word), reportlab(PDF)
- UI: Jinja2 + Vanilla JS
- 세션: JSON 파일 영속화

## 7. 활용 시나리오

- **개인 사업자/프리랜서**: 출장 영수증 분기별 정리 → 세무사에게 정제된 Excel 전달
- **중소기업 영업/CEO**: 임원 출장 정산 처리 시간 1/10로 단축
- **컨설턴트/연구원**: 다국가 출장의 통화별 환산·재무 보고
- **세무 검증 보조**: 매입세액 공제 자동 판정으로 세무 리스크 감소

## 8. 운영 결과 / 효과

- **시간 절감**: 영수증 30장 기준 수동 1.5~2시간 → 자동 5~10분
- **세무 정합성**: 공제 누락·오공제 자동 룰 검증 → 사람 실수 0
- **개인정보 보호**: OCR 전체가 로컬에서 동작 → 거래처 정보 외부 전송 없음
- **운영비 0원**: 클라우드 OCR 비용 0, 1회 환경 구축 후 영구 무료

## 9. 향후 개선 방향

- **신용카드 명세서 매칭** — 명세서 CSV 업로드 → 영수증과 자동 매칭으로 누락 영수증 탐지
- **환율 자동 적용** — 영수증 일자 기준 한국은행 고시환율 자동 조회·환산
- **법인카드 분리 처리** — 법인카드/개인카드 라벨 분리 → 회사 정산용 vs 개인 정산용
- **모바일 캡쳐 앱** — 카메라로 영수증을 찍자마자 서버에 업로드되는 PWA
- **세무사 SaaS 연동** — 더존·세무사랑 등에 직접 인입

## 10. 한계 / 알려진 이슈

- **OCR 정확도**: EasyOCR은 클라우드 OCR(Google Vision 등) 대비 정확도가 낮다 → 일부 영수증은 수동 보정 필요. 파싱 결과는 항상 사용자가 한 번 검수.
- **첫 실행 지연**: EasyOCR 모델 다운로드(약 100MB)로 최초 실행 시 1~2분 대기.
- **GPU 미사용 시 속도**: CPU만 사용 시 영수증 1장당 3~10초. GPU 환경이면 0.3~1초.
- **PDF 다중 페이지**: 다중 페이지 PDF는 페이지마다 별도 영수증으로 처리됨 (병합 영수증은 미지원).
- **세무 룰**: 한국 세법 기준. 해외법인·미국 IRS 등 타국 세무는 미지원.
- **세션 동시성**: 단일 사용자 로컬 도구로 설계 → 멀티유저 동시 사용 시 race condition 가능.
